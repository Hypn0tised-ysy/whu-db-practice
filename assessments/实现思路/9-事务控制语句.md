# 任务9：事务控制语句（BEGIN/COMMIT/ABORT）

## 提交记录
`daffb42` - task9: transaction

## 问题分析
框架将每条SQL语句包装为独立隐式事务并自动提交。需要支持：
1. 显式事务（BEGIN...COMMIT/ABORT跨多条语句）
2. 事务回滚（ABORT时撤销事务内所有写操作）
3. WriteRecord追踪（INSERT/DELETE/UPDATE记录用于回滚）

## 修改文件

| 文件 | 改动内容 |
|------|---------|
| `src/transaction/transaction_manager.cpp` | `begin()`实现（已有一部分），`commit()`设置状态，`abort()`实现完整回滚逻辑 |
| `src/execution/executor_insert.h` | 插入成功后创建WriteRecord(INSERT_TUPLE) |
| `src/execution/executor_delete.h` | 删除前保存旧值，创建WriteRecord(DELETE_TUPLE) |
| `src/execution/executor_update.h` | 更新前保存旧值，创建WriteRecord(UPDATE_TUPLE) |

## 实现思路

### 1. 事务生命周期
- **BEGIN**: `context->txn_->set_txn_mode(true)` — 标记为显式事务，后续语句复用同一事务
- **SQL语句**: 在显式事务中执行，写操作创建WriteRecord
- **COMMIT**: 设置状态为COMMITTED，清空write_set
- **ABORT**: 逆序遍历write_set，撤销操作→设置状态为ABORTED

### 2. WriteRecord
三种类型的回滚信息：
- INSERT_TUPLE: 存储tab_name + rid → 回滚时删除该记录+索引
- DELETE_TUPLE: 存储tab_name + rid + 旧记录数据 → 回滚时重新插入旧记录+索引
- UPDATE_TUPLE: 存储tab_name + rid + 旧记录数据 → 回滚时恢复旧记录+索引

### 3. 回滚逻辑（abort）
```cpp
while (!write_set.empty()) {
    wr = write_set.back(); write_set.pop_back();
    if (INSERT_TUPLE): fh->delete_record(rid) + 删除索引条目
    if (DELETE_TUPLE): fh->insert_record(rid, old_data) + 重新插入索引
    if (UPDATE_TUPLE): fh->update_record(rid, old_data) + 更新索引(delete新, insert旧)
    delete wr;
}
```

### 4. 关键测试
显式事务需要单持久TCP连接（pipe所有SQL到同一个client）。每个echo创建新连接会导致不同线程处理，事务无法跨语句。
