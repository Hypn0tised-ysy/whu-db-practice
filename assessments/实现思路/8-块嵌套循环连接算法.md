# 任务8：块嵌套循环连接算法（Block Nested-Loop Join）

## 提交记录
`1198958` - task8: nested loop join

## 问题分析
原NestedLoopJoinExecutor是逐行（row-by-row）的连接，每次外层取一行、内层全表扫描。需要改为块嵌套循环：将外层多行缓存到join buffer，内层一次扫描匹配buffer中所有行。

## 修改文件

| 文件 | 改动内容 |
|------|---------|
| `src/execution/executor_nestedloop_join.h` | 重写为Block NLJ：增加`left_block_`缓冲区、`JOIN_BUFFER_ROWS=500`、`fill_left_block()` |

## 实现思路

### 1. 块缓冲区设计
- `JOIN_BUFFER_ROWS = 500`：每次缓冲最多500行左表数据
- `left_block_: vector<vector<char>>`：存储左表行数据拷贝
- `fill_left_block()`：从child执行器读取`JOIN_BUFFER_ROWS`行到缓冲区

### 2. 执行算法
```
beginTuple():
  1. left_->beginTuple()
  2. fill_left_block() - 读第一批左表行
  3. right_->beginTuple()
  4. 找到第一个匹配对（遍历right行 × 所有buffered left行）

nextTuple():
  1. right_->nextTuple() 前进右表
  2. 若右表到末尾：
     block_idx_++ （尝试下一个buffered左行）
     若所有buffered左行都遍历完：fill_left_block()读下一批
     重置右表
  3. 找到下一个匹配对

Next():
  从left_block_[block_idx_]取左表数据 + right_->Next()取右表数据 → 合并返回
```

### 3. 条件比较
- `check_current_pair()`：对当前left+right组合，按连接条件逐列比较
- 支持非等值连接（<, >, <=, >=, <>, =）
- 支持所有数据类型（INT/BIGINT/FLOAT/STRING/DATETIME）

### 4. 性能优化
- 原NLJ复杂度O(N×M)：对每个左表行扫描整个右表
- BNLJ复杂度O(N×M/K)：K=JOIN_BUFFER_ROWS时，右表扫描次数减少K倍
