# 任务10：基于死锁预防的可串行化隔离级别（2PL + No-Wait）

## 提交记录
`a20d246` - task10: 2PL and no-wait algorithm

## 问题分析
框架的LockManager所有加锁函数均为空实现（return true）。需要实现：
1. 两阶段封锁协议（2PL）保证可串行化
2. 多粒度锁（表级IS/IX/S/SIX/X + 行级S/X）
3. No-Wait死锁预防（不兼容立即abort）
4. 锁在COMMIT/ABORT时统一释放（shrinking phase）

## 实现思路

### 1. 锁兼容矩阵
```
      NL  IS  IX  S  SIX  X
NL    Y   Y   Y   Y   Y   Y
IS    Y   Y   Y   Y   Y   N
IX    Y   Y   Y   N   N   N
S     Y   Y   N   Y   N   N
SIX   Y   Y   N   N   N   N
X     Y   N   N   N   N   N
```

### 2. No-Wait死锁预防
- 请求锁时先检查兼容性
- 不兼容→抛出`TransactionAbortException`（立即回滚，不等待）
- 锁升级（S→X）时检查是否唯一持有者
- 等待队列中有pending请求→abort（no-wait策略不排队）

### 3. 两阶段封锁
- **Growing phase**: 按需获取锁，`lock()`成功后添加到`txn->get_lock_set()`
- **Shrinking phase**: COMMIT/ABORT时遍历`lock_set`调用`unlock()`→清空

### 4. 多粒度锁
- 表级：IS（意向共享）、IX（意向排他）、S（共享）、SIX（S+IX）、X（排他）
- 行级：S（共享读）、X（排他写）
- LockDataId统一标识：表级（fd, TABLE），行级（fd, rid, RECORD）

### 5. 脏读预防（测试示例）
- Txn1(UPDATE row2)→X锁; Txn2(SELECT row2)→S锁冲突→No-Wait abort
- Txn2收到"abort\n"→客户端识别回滚
- Txn1 ABORT→释放锁+回滚修改→数据一致性保持
